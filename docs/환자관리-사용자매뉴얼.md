## 1. 시스템 접속

### 접속 방법
1. 웹 브라우저(Chrome 권장)를 실행합니다
2. 주소창에 시스템 URL을 입력합니다
3. 환자 관리 페이지로 이동: 좌측 사이드바에서 **"환자 관리"** 클릭

---

## 2. 화면 구성

환자 관리 페이지는 다음과 같이 구성되어 있습니다:

```
┌─────────────────────────────────────────────────────────────┐
│  환자 관리          총 OOO명    [기간 필터 ▼]    [+ 환자 등록] │  ← 헤더
├─────────────────────────────────────────────────────────────┤
│  [노쇼/지연 O명]  [오늘 예정 O명]  [장기 방치 O명]              │  ← 긴급 요약 카드
├─────────────────────────────────────────────────────────────┤
│  전체 | 전화상담 | 내원예약 | 내원완료 | ... | 사후관리  [검색] │  ← 상태 필터 탭
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  환자 목록 (테이블)                                          │  ← 환자 리스트
│                                                             │
├─────────────────────────────────────────────────────────────┤
│  OOO명                                        < 1 2 3 4 5 > │  ← 페이지네이션
└─────────────────────────────────────────────────────────────┘

[CTI 패널] ← 좌측 하단 (전화 수신 시 팝업)
```

## 3. 기간 필터 사용법

화면 상단의 **기간 필터**를 사용하여 조회할 환자의 범위를 설정합니다.

### 프리셋 옵션
| 옵션 | 설명 |
|------|------|
| 최근 1개월 | 최근 1개월 내 등록된 환자만 표시 |
| **최근 3개월** | 기본값. 최근 3개월 내 등록된 환자 표시 |
| 최근 6개월 | 최근 6개월 내 등록된 환자 표시 |
| 최근 1년 | 최근 1년 내 등록된 환자 표시 |
| 전체 | 모든 환자 표시 (데이터가 많으면 느려질 수 있음) |

### 직접 날짜 선택
특정 기간을 조회하고 싶을 때:
1. 기간 필터 버튼 클릭
2. 하단의 **"직접 선택"** 영역에서 시작일/종료일 입력
3. **[적용]** 버튼 클릭

> **팁**: 데이터가 많이 쌓이면 "전체"보다는 기간을 지정하는 것이 빠릅니다.

## 4. 긴급 환자 관리

화면 상단의 **긴급 요약 카드**로 즉시 조치가 필요한 환자를 빠르게 파악할 수 있습니다.

### 긴급 카드 종류

| 카드 | 색상 | 의미 | 조치 필요 |
|------|------|------|----------|
| **노쇼/지연** | 🔴 빨강 | 예약일이 지났는데 내원하지 않은 환자 | 즉시 연락 필요 |
| **오늘 예정** | 🔵 파랑 | 오늘 예약이 있는 환자 | 내원 확인 필요 |
| **장기 방치** | 🟡 노랑 | 오래 연락 없이 방치된 환자 | 팔로업 연락 필요 |

### 사용 방법
- 카드를 **클릭**하면 해당 유형의 환자만 필터링됩니다
- 다시 클릭하면 필터가 해제됩니다

### 환자 목록에서 표시
긴급 환자는 목록에서 **좌측 색상 바**로 구분됩니다:
- 🔴 빨간 바: 노쇼/지연
- 🔵 파란 바: 오늘 예정
- 🟡 노란 바: 장기 방치

## 5. 환자 상태(퍼널) 이해하기

환자는 다음 순서로 상태가 변경됩니다:

```
전화상담 → 내원예약 → 내원완료 → 치료예약 → 치료중 → 치료완료 → 사후관리
```

### 각 상태 설명

| 상태 | 설명 | 다음 액션 |
|------|------|----------|
| **전화상담** | 첫 전화 상담 진행 중 | 내원 예약 잡기 |
| **내원예약** | 내원 예약이 잡힌 상태 | 예약일에 내원 확인 |
| **내원완료** | 치과에 방문하여 상담 완료 | 치료 계획 수립 |
| **치료예약** | 치료 일정이 잡힌 상태 | 치료 시작 |
| **치료중** | 현재 치료 진행 중 | 치료 완료까지 관리 |
| **치료완료** | 모든 치료 완료 | 사후관리 전환 |
| **사후관리** | 치료 후 정기 관리 | 정기 연락 |

### 상태별 체류일 경고
각 상태에서 너무 오래 머무르면 **"장기 방치"**로 표시됩니다:
- 전화상담: 7일 이상
- 내원완료: 7일 이상
- 치료중: 30일 이상
- 사후관리: 90일 이상

## 5-1. 환자 목록 테이블 읽는 법

환자 목록의 각 열(컬럼)이 의미하는 바를 이해해야 효율적으로 환자를 관리할 수 있습니다.

### 테이블 컬럼 구성

| 컬럼 | 설명 | 예시 |
|------|------|------|
| **유형** | 마지막 통화가 수신/발신인지 표시 | 🟢 초록=수신, 🔵 파랑=발신 |
| **환자명** | 환자 이름 + 유입경로 | 김철수 (네이버광고) |
| **전화번호** | 연락처 | 010-1234-5678 |
| **관심시술** | 관심있는 치료 항목 | 임플란트 |
| **상태** | 현재 진행 단계 | 전화상담 |
| **관심도** | 치료 의향 정도 | 🔥높음 / 🌡️중간 / ❄️낮음 |
| **다음 일정** | 다음 액션 + D-day **(중요!)** | 아래 참조 |

### ⭐ "다음 일정" 컬럼 읽는 법 (가장 중요!)

다음 일정 컬럼은 환자별로 **지금 무엇을 해야 하는지** 한눈에 보여줍니다.

| 표시 형식 | 의미 | 조치 |
|----------|------|------|
| **내원예약 D-Day** (파랑, 굵게) | 오늘이 예약일! | ⚡ 내원 여부 즉시 확인 |
| **내원예약 D-3** (주황) | 3일 후 예약 | 리마인드 연락 고려 |
| 내원예약 D-7 (회색) | 7일 후 예약 | 일정 확인만 |
| **내원예약 +2일** (빨강) | 예약일 2일 지남 (노쇼!) | ⚡ 즉시 연락 필요! |
| 5일째 (회색) | 현재 상태에서 5일 경과 | 다음 액션 설정 필요 |
| **8일째** (주황) | 주의! 오래 방치됨 | 팔로업 연락 필요 |
| **15일째** (빨강) | 위험! 장기 방치 | ⚡ 즉시 연락! |

### 행(Row) 색상으로 긴급도 파악하기

환자 행의 좌측에 색상 바가 있으면 **긴급한 환자**입니다:

| 좌측 바 색상 | 배경색 | 의미 | 조치 |
|-------------|--------|------|------|
| 🔴 빨간 바 | 연한 빨강 | 노쇼/지연 - 예약일이 지났음 | 1순위로 연락! |
| 🔵 파란 바 | 연한 파랑 | 오늘 예정 - 오늘 예약 | 내원 확인! |
| 🟡 노란 바 | 연한 노랑 | 장기 방치 - 오래 연락 없음 | 팔로업 연락! |
| 없음 | 흰색 | 정상 - 긴급하지 않음 | 일반 관리 |

### 관심도(온도) 아이콘

| 아이콘 | 의미 | 설명 |
|--------|------|------|
| 🔥 높음 | Hot | 치료 의향 높음, 적극적으로 상담 |
| 🌡️ 중간 | Warm | 관심은 있으나 고민 중 |
| ❄️ 낮음 | Cold | 관심도 낮음, 장기 관리 필요 |

> **팁**: 매일 아침 출근하면 **"오늘 예정"** 카드를 먼저 클릭하고, 그 다음 **"노쇼/지연"** 카드를 클릭해서 순서대로 처리하세요!

## 6. 환자 검색 및 필터링

### 검색
화면 우측 상단의 검색창에서 환자를 검색할 수 있습니다:
- **이름**으로 검색: 예) "김철수"
- **전화번호**로 검색: 예) "010-1234" 또는 "1234"

### 상태 필터
상태 탭을 클릭하여 특정 상태의 환자만 볼 수 있습니다:
- **전체**: 모든 환자
- **전화상담**: 상담 중인 환자만
- **내원예약**: 예약된 환자만
- ... (각 상태별 필터)

> **참고**: 상태 필터와 긴급 필터는 동시에 사용할 수 없습니다. 하나를 선택하면 다른 하나는 자동 해제됩니다.

## 7. 환자 등록

### 신규 환자 등록
1. 화면 우측 상단의 **[+ 환자 등록]** 버튼 클릭
2. 환자 정보 입력:
   - 이름 (필수)
   - 전화번호 (필수)
   - 관심 분야
   - 유입 경로
   - 다음 액션
3. **[등록]** 버튼 클릭

### 자동 등록 (CTI 연동)
전화가 오면 자동으로 신규 환자가 등록됩니다:
- 기존 환자: 기존 정보로 연결
- 신규 환자: 전화번호로 자동 등록 후 정보 보완 필요

## 8. 환자 상세 정보

환자 목록에서 환자를 클릭하면 상세 페이지로 이동합니다.

### 상세 페이지 구성
- **기본 정보**: 이름, 전화번호, 등록일
- **상태 관리**: 현재 상태 및 변경 이력
- **통화 기록**: 수신/발신 통화 목록
- **AI 분석**: 통화 내용 자동 분석 결과
- **메모**: 상담 메모 기록

### 상태 변경
1. 상세 페이지에서 **상태 변경** 버튼 클릭
2. 새로운 상태 선택
3. 필요시 다음 예약일 입력
4. 저장

## 9. CTI 패널 (전화 연동)

화면 좌측 하단에 **CTI 패널**이 있습니다.

### 기본 상태
- 작은 버튼 형태로 표시
- 클릭하면 패널이 확장됨

### 전화 수신 시
1. 패널이 자동으로 확장되며 파란색으로 깜빡임
2. 발신자 정보 표시:
   - 기존 환자: 이름과 상태 표시
   - 신규 환자: 전화번호만 표시, 자동 등록
3. **[환자 상세 보기]** 클릭 시 해당 환자 페이지로 이동

### 최근 통화 목록
패널 하단에 최근 5건의 통화가 표시됩니다.

### AI 분석
통화 종료 후 AI가 자동으로 통화 내용을 분석합니다:
- 환자 온도(관심도) 자동 판정
- 통화 요약 생성
- 보라색 배경으로 표시

## 10. 자주 묻는 질문 (FAQ)

### Q: 환자가 검색되지 않아요
**A**: 다음을 확인해보세요:
1. 기간 필터가 너무 좁게 설정되어 있지 않은지 확인
2. "전체" 기간으로 변경 후 다시 검색
3. 전화번호 검색 시 하이픈(-) 없이 숫자만 입력해보기

### Q: 긴급 환자 카드 숫자가 0인데 환자가 있어요
**A**: 긴급 환자는 다음 조건을 만족해야 표시됩니다:
- 노쇼/지연: 예약일(nextActionDate)이 과거인 경우
- 오늘 예정: 예약일이 오늘인 경우
- 장기 방치: 상태별 임계일을 초과한 경우

### Q: CTI 패널이 "연결 안됨"으로 표시돼요
**A**: CTI 서버와 연결이 끊어진 상태입니다:
1. 페이지 새로고침 시도
2. 계속 안되면 관리자에게 문의

### Q: 전화가 왔는데 CTI에 표시가 안 돼요
**A**: 다음을 확인해보세요:
1. CTI 패널이 "연결됨" 상태인지 확인
2. 브라우저 탭이 활성화되어 있는지 확인
3. CTI Bridge 프로그램이 실행 중인지 확인 (관리자 문의)

### Q: 환자 상태를 잘못 변경했어요
**A**: 환자 상세 페이지에서 다시 올바른 상태로 변경하면 됩니다. 상태 변경 이력은 자동 저장됩니다.
